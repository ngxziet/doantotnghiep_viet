Main loop — vòng lặp chính

Firmware chạy trên 1 core duy nhất, không dùng RTOS task. Mọi thứ chạy tuần tự trong loop():

main.cpp
// Chạy LIÊN TỤC — không có delay
void loop() {
    // 1. Kiểm tra WiFi station (mỗi 5 giây)
    // 2. Nhận lệnh UDP từ app (chạy MỌI vòng lặp)
    // 3. Rate limit: bỏ qua nếu chưa đủ 20ms kể từ lần xử lý trước
    if (now - lastLoopMs < 20) return;  // ← 50Hz control loop
    // 4. Đọc IMU + encoder
    // 5. Cập nhật odometry (vị trí xe)
    // 6. Chạy navigator (state machine điều khiển)
    // 7. Gửi telemetry về app (mỗi 100ms)
    // 8. In debug ra Serial (mỗi 500ms)
}
Tại sao 50Hz? LOOP_INTERVAL_MS = 20ms → 50 lần/giây. Đủ nhanh để phản ứng kịp khi xe chạy tốc độ vừa phải (175 PWM ≈ 30cm/s). Nhận lệnh UDP chạy MỌI iteration (không rate limit) để không bỏ lỡ packet.

STEP 2 Encoder (LM393)

Encoder là gì? Bộ đếm xung gắn trên bánh xe. Mỗi khi bánh quay, encoder phát ra xung. Đếm số xung = biết bánh quay bao nhiêu vòng = biết xe đi bao xa.

Thông số phần cứng

Thông sốGiá trịÝ nghĩa
Xung/vòng20Bánh quay 1 vòng = 20 xung
Đường kính bánh6.5 cmChu vi = π × 6.5 = 20.42 cm
cm/xung1.021 cm20.42 / 20 ≈ 1 cm mỗi xung
Xung/node4950cm (khoảng cách 1 node) / 1.021 cm ≈ 49 xung

Cách hoạt động — Interrupt driven

encoder.cpp
// ISR (Interrupt Service Routine) — chạy khi encoder phát xung RISING
// Cực kỳ nhanh, chạy trong ngữ cảnh interrupt
void IRAM_ATTR Encoder::onRisingA() {
    unsigned long now = micros();

    // Lọc nhiễu: bỏ qua xung cách nhau < 1500µs
    // Tốc độ max = 666 xung/giây (thực tế ~200 xung/s ở tốc độ cao nhất)
    if (now - _lastPulseMicros < 1500) return;
    _lastPulseMicros = now;

    // _direction được set từ lệnh motor:
    //   +1 khi motor quay tiến
    //   -1 khi motor quay lùi
    //    0 khi motor dừng (bỏ qua xung nhiễu lúc dừng)
    _pulseCount += _direction;
}
Tại sao cần debounce 1500µs? Encoder LM393 giá rẻ hay bị nhiễu EMI từ motor. Khi motor chạy, nhiễu điện từ tạo ra xung giả. 1500µs debounce lọc hết nhiễu mà vẫn cho phép đến 666 xung/giây (gấp 3 lần tốc độ max thực tế).

Đọc xung an toàn — atomic read

encoder.cpp
long Encoder::getPulseCount() {
    noInterrupts();       // Tắt interrupt tạm thời
    long count = _pulseCount; // Đọc giá trị
    interrupts();         // Bật interrupt lại
    return count;
    // Tại sao? Vì _pulseCount có thể bị ISR thay đổi giữa chừng
    // khi đang đọc → giá trị bị sai (race condition)
}

STEP 3 IMU MPU6050

IMU là gì? Inertial Measurement Unit — cảm biến đo gia tốc + vận tốc góc. Trong project này chỉ dùng gyroscope trục Z (con quay hồi chuyển) để đo hướng xoay của xe.

Nguyên lý đo hướng

Gyro Z đo: vận tốc góc (°/giây) │ ▼ tích phân theo thời gian heading += gyroZ × dt │ ▼ Hướng tuyệt đối (radian) ← đây là thứ ta cần

Calibration — loại bỏ drift

imu_mpu6050.cpp
void ImuMpu6050::calibrate(int samples) {
    // Khi xe ĐỨNG YÊN, đo gyro Z nhiều lần
    // Gyro lý tưởng sẽ đọc = 0, nhưng thực tế luôn có bias (lệch)
    // Ví dụ: thay vì 0, nó đọc 3.5 liên tục → bias = 3.5

    for (int i = 0; i < samples; i++) {
        // Đọc raw gyro Z
        // 20 mẫu đầu: lấy hết (chưa đủ data để lọc)
        // Từ mẫu 21: chỉ lấy nếu gần trung bình (loại outlier)
        // → _gyroBiasZ = trung bình các mẫu "sạch"
    }

    // Sau calibration: heading = tích phân (gyroZ - bias) × dt
    // Bias bị trừ đi → heading không bị trôi khi xe đứng yên
}

Update — tích phân gyro mỗi loop

imu_mpu6050.cpp
bool ImuMpu6050::update() {
    float dt = (now - _lastUpdateMs) / 1000.0f;  // thời gian giữa 2 lần đo (giây)

    // Đọc raw gyro Z từ I2C
    // Chuyển sang rad/s: (raw - bias) × GYRO_SCALE
    float gyroZ_rads = (gz - _gyroBiasZ) * GYRO_SCALE_RAD_S;
    //                                        ↑
    // GYRO_SCALE = 1 / (131 × 180/π) ≈ 0.000133 rad/s per LSB
    // MPU6050 ở chế độ ±250°/s: 131 LSB = 1°/s

    // Deadband: bỏ qua gyro < 0.01 rad/s (≈ 0.57°/s)
    // → loại nhiễu nhỏ khi xe đứng yên hoặc rung nhẹ
    if (fabsf(gyroZ_rads) < 0.01) gyroZ_rads = 0.0f;

    // Tích phân: heading += vận tốc góc × thời gian
    _yawRad += gyroZ_rads * dt;

    // Chuẩn hóa về [-π, π]
    _yawRad = _normalizeAngle(_yawRad);
}

Reference angle — hướng gốc

imu_mpu6050.cpp
// getAbsoluteYawRad() trả về hướng TƯƠNG ĐỐI so với lúc calibrate
// Ví dụ: calibrate xong heading = 0°, xe xoay phải 45° → trả về -45°
float ImuMpu6050::getAbsoluteYawRad() const {
    return _normalizeAngle(_yawRad - _referenceYawRad);
    //                      ↑ tích lũy    ↑ mốc gốc lúc calibrate
}
Gyro drift Gyro MPU6050 sẽ drift theo thời gian (heading trôi dần dù xe đứng yên). Calibration + deadband giảm drift nhưng không loại hết. Đó là lý do firmware dùng complementary filter kết hợp encoder + IMU trong odometry.

STEP 4 Odometry — Ước lượng vị trí

Odometry là gì? Kỹ thuật ước lượng vị trí (x, y) và hướng (theta) của xe dựa trên encoder + IMU. Kết quả là Pose {x, y, theta} — vị trí hiện tại của xe trên bản đồ.

Công thức cốt lõi — Differential Drive

Encoder trái đếm: deltaLeft xung Encoder phải đếm: deltaRight xung │ ▼ chuyển sang mét dLeft = deltaLeft × CM_PER_PULSE / 100 dRight = deltaRight × CM_PER_PULSE / 100 │ ▼ tính quãng đường trung bình + góc xoay d = (dLeft + dRight) / 2 ← quãng đường đi thẳng dTheta = (dRight - dLeft) / wheelBase ← góc xoay (từ encoder) │ ▼ cập nhật vị trí x += d × cos(theta + dTheta/2) y -= d × sin(theta + dTheta/2) ← trừ vì trục Y hướng xuống theta += dTheta

Complementary Filter — kết hợp encoder + IMU

odometry.cpp
// Alpha = 0.35 → IMU chiếm 35%, encoder chiếm 65%
float thetaEnc = _normalizeAngle(_pose.theta + dThetaEnc);
float thetaFused = thetaEnc;

if (useImuHeading) {
    // Tính sai lệch giữa IMU và encoder
    float imuError = _normalizeAngle(imuYawRad - thetaEnc);

    // Trộn: encoder + 35% × (IMU - encoder)
    // → encoder nhanh + IMU ổn định = kết quả tốt hơn cả 2
    thetaFused = _normalizeAngle(thetaEnc + 0.35 * imuError);
}

Suppress Translation — khi xoay tại chỗ

odometry.cpp
if (suppressTranslation) {
    // Khi xe XOAY TẠI CHỖ:
    // - 2 bánh quay ngược nhau → encoder đếm xung
    // - Nhưng xe KHÔNG DI CHUYỂN (x, y không đổi)
    // → Chỉ cập nhật theta, KHÔNG cập nhật x, y
    // Nếu không suppress: xe xoay → odometry nghĩ xe đi thẳng → vị trí sai
    _pose.theta = imuYawRad;  // chỉ dùng IMU cho heading

}

Override heading khi đi thẳng

main.cpp

STEP 5 Motor Driver (L298N)

L298N là gì? Mạch điều khiển motor DC. ESP32 gửi tín hiệu PWM (0-255) → L298N khuếch đại → motor quay nhanh/chậm. 2 chân IN1/IN2 quyết định chiều quay, chân ENA quyết định tốc độ.

Điều khiển 1 motor

motor_driver.cpp
void MotorDriver::_applyLeft(int speed) {
    speed = _clamp(speed, -255, 255);
    _leftSpeed = speed;

    if (speed > 0) {
        digitalWrite(_in1, HIGH);  // IN1=HIGH, IN2=LOW → quay tiến
        digitalWrite(_in2, LOW);
    } else if (speed < 0) {
        digitalWrite(_in1, LOW);   // IN1=LOW, IN2=HIGH → quay lùi
        digitalWrite(_in2, HIGH);
    } else {
        digitalWrite(_in1, LOW);   // cả 2 LOW → dừng (coast)
        digitalWrite(_in2, LOW);
    }
    ledcWrite(MOTOR_CH_LEFT, abs(speed));  // PWM 0-255
}

Slew Rate Limiting — tăng tốc từ từ

motor_driver.cpp
// Thay vì nhảy đột ngột từ 100 lên 200 PWM:
// → tăng từng bước tối đa 18 PWM mỗi loop (mỗi 20ms)
// → motor không bị giật, encoder không bị lỡ xung
void MotorDriver::setLeftSlew(int target, int maxStep) {
    int step = _clamp(target - _leftSpeed, -maxStep, maxStep);
    _applyLeft(_leftSpeed + step);
    // Ví dụ: hiện tại 100, target 200, maxStep 18
    // Loop 1: 100 → 118
    // Loop 2: 118 → 136
    // Loop 3: 136 → 154
    // ... đến khi đạt 200
}

STEP 7 Navigation — Thuật toán xoay 2 pha

Xoay 2 pha: nhanh trước, tinh chỉnh sau Với góc lớn (>25°), xe xoay liên tục theo thời gian ước lượng để gần đích nhanh (pha 1), rồi dùng nudge burst để tinh chỉnh phần còn lại (pha 2). Với góc nhỏ (10°-25°), chỉ dùng nudge.

Pha 1 — Xoay nhanh (FastRotate) cho góc > 25°

node_navigator.cpp — _beginRotateOrDrive()
// Ước lượng thời gian xoay dựa trên tốc độ đã biết
float angleDeg = headingError * (180 / π);
float estimatedMs = (angleDeg / 130.0) * 1000;
//                               ↑ FAST_ROTATE_SPEED_DPS = 130°/s

estimatedMs *= 0.75;  // chỉ xoay 75% thời gian → tránh trượt quá

// Ví dụ xoay 90°:
//   estimatedMs = (90 / 130) × 1000 × 0.75 = 519ms
//   → quay liên tục PWM 180 trong 519ms
//   → đạt khoảng 67° (75% của 90°)
//   → còn lại ~23° → nudge xử lý tiếp
node_navigator.cpp — _handleFastRotate()
// Trong pha xoay nhanh, motor quay liên tục ở PWM 180
// Kiểm tra 3 điều kiện dừng:

// 1. Timeout bảo vệ 9 giây → fault
// 2. Đã gần đích (≤ 4°) → dừng sớm, không cần nudge
// 3. Hết thời gian ước lượng → dừng, chuyển settle

// Sau khi dừng → RotateSettle → đo heading
// → nếu còn lệch > 4° → chuyển sang Rotate (nudge)

Pha 2 — Nudge tinh chỉnh (góc nhỏ hoặc phần còn lại)

Nudge rotation là gì? Thuật toán xoay từng burst ngắn → dừng → đo → lặp lại. Giống như vặn ốc từng cái nhẹ, kiểm tra sau mỗi lần vặn. Dùng cho góc 10°-25° hoặc phần dư sau FastRotate.

Vòng lặp Nudge

node_navigator.cpp — _handleRotate()
// ┌─────────────────────────────────────────────────────────────┐
// │  BƯỚC 1: Kiểm tra đã đủ gần chưa                          │
// └─────────────────────────────────────────────────────────────┘
float error = _normalizeAngle(_targetHeading - pose.theta);
if (fabsf(error) <= 4°) {
    motors.stop();
    // → chuyển sang RotateSettle để xác nhận
}

// ┌─────────────────────────────────────────────────────────────┐
// │  BƯỚC 2: Timeout bảo vệ (9 giây)                          │
// └─────────────────────────────────────────────────────────────┘
if (elapsed > 9000ms) {
    // Xoay quá lâu → fault → dừng hẳn
    // Nguyên nhân: motor yếu, bánh kẹt, IMU hỏng
}

// ┌─────────────────────────────────────────────────────────────┐
// │  BƯỚC 3: Burst đã hết 50ms → dừng, chờ ổn định           │
// └─────────────────────────────────────────────────────────────┘
if (elapsed_burst >= 50ms) {
    motors.stop();
    // → RotateSettle, chờ 150ms rồi đo lại heading
}

// ┌─────────────────────────────────────────────────────────────┐
// │  BƯỚC 4: Trong burst → quay motor                         │
// └─────────────────────────────────────────────────────────────┘
int dir = (error > 0 ? 1 : -1);
// error > 0: cần xoay trái (ngược chiều kim đồng hồ)
// error < 0: cần xoay phải (chiều kim đồng hồ)
motors.setLeft(-pwm × dir);   // 2 bánh quay ngược nhau
motors.setRight(pwm × dir);

Adaptive PWM — tự tăng lực nếu xoay không nhúc nhích

node_navigator.cpp — _handleRotateSettle()
// Sau mỗi burst, đo xem heading thay đổi bao nhiêu
float progress = fabsf(pose.theta - _headingBeforeNudge);

if (progress < 1°) {
    // Heading hầu như không đổi → mặt sàn trơn / motor yếu
    // → Tăng PWM thêm 10, tối đa 220
    _nudgePwm += 10;  // 160 → 170 → 180 → ... → 220
}

// Rồi quay lại State::Rotate, thử burst mới với PWM cao hơn
Tại sao burst 50ms + settle 150ms?
  • 50ms burst: đủ để motor tạo moment xoay, nhưng ngắn để không trượt quá
  • 150ms settle: chờ xe dừng hẳn (quán tính), IMU ổn định, rồi mới đo heading chính xác
  • Nếu burst quá dài → trượt quá → phải xoay ngược → dao động

STEP 8 Navigation — Thuật toán đi thẳng

Đây là thuật toán phức tạp nhất trong firmware. Xe vừa đi thẳng vừa phải liên tục chỉnh hướng bằng 2 loại correction kết hợp: heading (IMU) + balance (encoder).

Tổng quan 6 bước

node_navigator.cpp — _handleDrive()
// ═══════════════════════════════════════════════════════════
//  BƯỚC 1: Đọc tiến trình từ encoder
// ═══════════════════════════════════════════════════════════
long leftProgress  = encoder_trái - lúc_bắt_đầu;
long rightProgress = encoder_phải - lúc_bắt_đầu;
long avgProgress   = (left + right) / 2;

// ═══════════════════════════════════════════════════════════
//  BƯỚC 2: Kiểm tra đã đến node chưa
// ═══════════════════════════════════════════════════════════
if (avgProgress >= _targetPulses) {  // targetPulses = 49 (1 node)
    // ĐÃ ĐẾN → snap vị trí, chuyển node tiếp hoặc dừng
}

// ═══════════════════════════════════════════════════════════
//  BƯỚC 3: Tính tốc độ cơ sở (giảm dần khi gần đích)
// ═══════════════════════════════════════════════════════════
int remaining = _targetPulses - avgProgress;
int base = 175;  // tốc độ cruise
if (remaining < 12) {
    // Còn < 12 xung (≈ 12cm) → giảm tốc từ 175 xuống 135
    // → dừng chính xác, không trượt qua node
    base = map(remaining, 0, 12, 135, 175);
}

Bước 4 — Heading Correction (IMU)

Tại sao dùng raw IMU thay vì odometry? Heading correction làm 2 bánh quay khác tốc độ → encoder nghĩ xe đang xoay → odometry cập nhật theta sai → correction dựa trên theta sai → feedback loop sai thêm. Dùng raw IMU cắt đứt vòng lặp này. 
node_navigator.cpp — bước 4
// headingError = hướng_mục_tiêu - hướng_IMU_đo_được
float headingError = _normalizeAngle(_targetHeading - imuHeadingRad);

int headingCorrection = 0;
if (fabsf(headingError) > 2°) {  // deadband: bỏ qua lệch < 2°
    headingCorrection = constrain(
        (int)(headingError * 149.0),  // GAIN = 149
        -50, 50                       // clamp ±50 PWM max
    );
}

// Bảng correction thực tế:
// ┌──────────┬────────────┬──────────────────────────┐
// │ Lệch (°) │  PWM chỉnh │ Giải thích               │
// ├──────────┼────────────┼──────────────────────────┤
// │    1°    │     0      │ Trong deadband, bỏ qua   │
// │    2°    │     5      │ Chỉnh nhẹ                │
// │    3°    │     8      │ 3 × π/180 × 149 = 7.8   │
// │    5°    │    13      │ 5 × π/180 × 149 = 13.0  │
// │    8°    │    21      │ Chỉnh mạnh               │
// │   10°    │    26      │ Gần max (ngưỡng xoay)    │
// └──────────┴────────────┴──────────────────────────┘

Bước 5 — Balance Correction (Encoder)

node_navigator.cpp — bước 5
// Nếu bánh phải quay nhiều hơn bánh trái → xe lệch trái
// → Giảm tốc phải, tăng tốc trái để cân bằng
int balanceCorrection = constrain(
    (int)((rightProgress - leftProgress) * 1.2),
    -50, 50
);

// Ví dụ: phải = 30 xung, trái = 27 xung
// balance = (30 - 27) × 1.2 = 3.6 → 4 PWM
// → tăng trái 4 PWM, giảm phải 4 PWM

Bước 6 — Kết hợp cả 2 correction

node_navigator.cpp — bước 6
// leftTarget  = base + balance - heading
// rightTarget = base - balance + heading
//
// Ví dụ: base=175, heading=+13 (lệch phải 5°), balance=+4 (phải nhanh hơn)
//
//   leftTarget  = 175 + 4 - 13 = 166 PWM
//   rightTarget = 175 - 4 + 13 = 184 PWM
//                 ↑              ↑
//            chậm hơn      nhanh hơn → xe quay phải để bù

int leftTarget  = constrain(base + balance - heading, 135, 220);
int rightTarget = constrain(base - balance + heading, 135, 220);

// Áp dụng motor trim (calibration tự động)
calibration.applyMotorTrim(&leftTarget, &rightTarget);

// Dùng slew rate (tăng từng 18 PWM/loop, không nhảy đột ngột)
motors.setLeftSlew(leftTarget, 18);
motors.setRightSlew(rightTarget, 18);

STEP 9 Navigation — Tối ưu đi thẳng liên tục

Cải tiến mới nhất. Trước đây xe dừng + xoay tại MỖI node. Giờ nếu heading lệch < 10° thì bỏ qua xoay, đi thẳng luôn.

Skip Rotate — bỏ qua xoay khi lệch nhỏ

node_navigator.cpp — _beginRotateOrDrive()
// Tính góc lệch giữa hướng hiện tại và hướng đến node tiếp
float headingError = fabsf(_normalizeAngle(_targetHeading - pose.theta));

if (headingError < 10°) {
    // Lệch nhỏ → heading correction khi đi thẳng sẽ tự bù
    // → Đi thẳng luôn, KHÔNG xoay
    _beginDrive(leftEncoder, rightEncoder);
    return;
}

// Lệch ≥ 10° → xoay trước (nudge rotation) rồi mới đi
_state = State::Rotate;

Collinear Drive — đi liên tục qua node thẳng hàng

}

STEP 10 Auto-Calibration

Tại sao cần calibration? 2 motor DC giá rẻ KHÔNG BAO GIỜ quay cùng tốc độ ở cùng PWM. Motor trái có thể nhanh hơn phải 5-10%. Calibration tự động đo và bù sai lệch này.

Motor Trim — tự chỉnh sau mỗi đoạn thẳng

calibration_manager.cpp — recordStraightSegment()
// Gọi SAU mỗi đoạn đi thẳng (1 node)
// So sánh encoder trái vs phải

float average = (leftPulses + rightPulses) * 0.5f;
float error = (rightPulses - leftPulses) / average;
// error > 0: phải quay nhiều hơn → phải nhanh hơn → cần giảm phải
// error < 0: trái quay nhiều hơn → trái nhanh hơn → cần giảm trái

// Tốc độ học: 5% mỗi đoạn thẳng → hội tụ sau ~5 đoạn
_leftTrim  += error * 0.05;   // tăng trim trái
_rightTrim -= error * 0.05;   // giảm trim phải

// Trim nằm trong [0.72, 1.28] → max ±28% bù
// Lưu vào NVS (flash) → giữ lại sau khi tắt nguồn
// Lần chạy đầu: lệch 3-4 đoạn rồi hội tụ
// Lần chạy sau: trim đã nhớ từ flash → thẳng ngay từ đoạn đầu

Áp dụng trim

calibration_manager.cpp — applyMotorTrim()
// Nhân PWM × trim trước khi gửi cho motor driver
// Ví dụ: leftTrim = 1.05, rightTrim = 0.95
// → PWM trái × 1.05 (nhanh hơn 5%)
// → PWM phải × 0.95 (chậm hơn 5%)
// → 2 bánh quay gần bằng nhau → xe đi thẳng

*leftPwm  = (int)(abs(*leftPwm)  * _leftTrim)  × sign;
*rightPwm = (int)(abs(*rightPwm) * _rightTrim) × sign;
NVS Persistence Calibration lưu vào NVS (Non-Volatile Storage) trên flash ESP32. Khi flash firmware mới với CONFIG_VERSION khác, calibration tự động reset. Trim tự lưu mỗi 15 giây (_dirty && millis() - _lastSaveMs >= 15000).

STEP 12 Main Loop chi tiết

main.cpp — loop()
void loop() {
    unsigned long now = millis();

    // ──── PHẦN 1: WiFi monitor (mỗi 5 giây) ────
    // Chỉ log, KHÔNG restart AP (tránh vòng lặp disconnect)

    // ──── PHẦN 2: Nhận lệnh UDP (MỌI vòng lặp) ────
    // UDP là fire-and-forget, phải poll liên tục
    UdpCommand command;
    if (udpServer.receiveCommand(routeXs, routeYs, &command)) {
        handleCommand(command);
    }

    // ──── PHẦN 3: Rate limit 50Hz ────
    if (now - lastLoopMs < 20) return;
    lastLoopMs = now;

    // ──── PHẦN 4: Đọc sensors ────
    imu.update();                    // đọc gyro Z, tích phân yaw
    long leftPulses  = encLeft.getPulseCount();
    long rightPulses = encRight.getPulseCount();

    // ──── PHẦN 5: Cập nhật odometry ────
    // suppressTranslation = true khi xoay tại chỗ
    // → chỉ update theta, không update x/y
    odometry.update(leftPulses, rightPulses, imuHeading,
                    imuConnected, suppressTranslation);

    // Override heading = pure IMU khi đi thẳng
    if (navigator.isDrivingStraight() && imuConnected) {
        odometry.setPose(p.x, p.y, imuHeading);
    }

    // ──── PHẦN 6: Điều khiển motor ────
    if (emergency) {
        // Ultrasonic phát hiện vật cản → dừng khẩn cấp
    } else if (manualMode) {
        // Điều khiển tay từ app → set motor trực tiếp
    } else {
        // Tự động: navigator state machine điều khiển
        navigator.update(pose, motors, encoders, calibration, imu);

        // Snap: khi đến node, đặt vị trí = tọa độ chính xác của node
        // → tránh tích lũy sai số odometry
        if (navigator.hasSnap()) {
            odometry.setPose(snapX, snapY, snapTheta);
        }
    }

    // ──── PHẦN 7: Cập nhật hướng encoder ────
    encLeft.setDirection(motors.getLeftSpeed());
    encRight.setDirection(motors.getRightSpeed());

    // ──── PHẦN 8: Gửi telemetry (mỗi 100ms) ────
    if (now - lastTelemetryMs >= 100) {
        udpServer.sendTelemetry(...);
    }

    // ──── PHẦN 9: Debug Serial (mỗi 500ms) ────
    if (now - lastDebugMs >= 500) {
        Serial.printf("pose=(...) status=... motor=.../...");
    }
}
Snap — chống tích lũy sai số Mỗi khi xe đến 1 node trên bản đồ, odometry được "snap" về tọa độ chính xác của node đó. Không có snap: sai số encoder tích lũy qua mỗi node → sau 10 node, vị trí lệch 5-10cm. Có snap: sai số reset về 0 tại mỗi node → chỉ sai trong 1 đoạn.
